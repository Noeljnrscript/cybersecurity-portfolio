ort in Cybersecurity

A port is a logical communication endpoint used by a computer to send or receive data over a network.

Think of it like this:

IP Address = the house address

Port Number = the specific room or door in the house

Application/Service = the person inside the room

So when data arrives at a computer:

The IP address identifies the device.

The port number identifies the specific application or service running on that device.

Example:

A web server might have the IP address:

192.168.1.10

But it can run many services:

Service	Port
Web server	80
Secure web server	443
File transfer	21
Email	25

The port number tells the system which service should handle the data.

2. Meaning of Port Numbers

A port number is a 16-bit numerical identifier used to identify a specific network service or process.

Port numbers range from:

0 – 65535

Each port allows communication between devices using networking protocols like:

TCP (Transmission Control Protocol)

UDP (User Datagram Protocol)

These protocols use port numbers to deliver data to the correct program.

Example:

When you open a website using HTTP, your browser communicates with the server through port 80.

3. Types of Port Number Ranges

Port numbers are divided into three main categories.

1. Well-Known Ports (0 – 1023)

These are reserved for common services and protocols.

Examples:

Port	Service
21	FTP (File Transfer Protocol)
22	SSH (Secure Shell)
23	Telnet
25	SMTP (Simple Mail Transfer Protocol)
53	DNS (Domain Name System)
80	HTTP (Hypertext Transfer Protocol)
443	HTTPS (Hypertext Transfer Protocol Secure)

These ports are standardized by the Internet Assigned Numbers Authority (IANA).

2. Registered Ports (1024 – 49151)

These ports are used by software applications and services.

Examples:

Port	Application
3306	MySQL
3389	Remote Desktop Protocol
8080	Alternative HTTP

Organizations can register these ports for their applications.

3. Dynamic / Private Ports (49152 – 65535)

These ports are also called:

Ephemeral ports

Temporary ports

They are used for temporary communication between devices.

Example:

When your browser connects to a website:

Server → Port 80

Your computer → random port like 52341

Once the session ends, the port closes.

4. Why Ports Are Important in Cybersecurity

Ports play a major role in network security monitoring and defense.

1. Controlling Network Traffic

Security devices like **Firewalls control which ports are open or closed.

Example:

Allow port 80 (web traffic)

Block port 23 (Telnet)

2. Detecting Cyber Attacks

Hackers often scan ports to find vulnerabilities using tools like:

Nmap

Metasploit

This technique is called port scanning.

If attackers find open ports, they may attempt:

unauthorized access

exploitation

malware injection

3. Securing Network Services

Cybersecurity professionals secure systems by:

Closing unused ports

Monitoring suspicious traffic

Using intrusion detection systems

Implementing strict firewall rules

5. Example of Port Communication

When you visit a secure website:

Your browser sends a request.

The request goes to the server's IP address.

It connects to port 443 using HTTPS.

The server sends the encrypted webpage back.

So communication looks like:

Client: 192.168.1.5:52341
Server: 142.250.190.78:443

Since you're studying networking in cybersecurity, the next important concepts after ports and port numbers are open ports, port scanning, and port forwarding. These are widely used in ethical hacking, penetration testing, and SOC monitoring.

1. Open Ports in Cybersecurity

An open port is a port on a computer or server that is actively accepting connections and communicating with other devices on a network.

When a port is open:

A service or application is running

The system is listening for incoming connections

Example:

If a web server is running:

Port 80 → open for HTTP (Hypertext Transfer Protocol)

Port 443 → open for HTTPS (Hypertext Transfer Protocol Secure)

This means users can connect to the website.

Example of Open Ports
Port	Service	Status
80	HTTP	Open
22	SSH	Open
21	FTP	Closed

If a port is closed, the system rejects connection attempts.

Why Open Ports Matter in Cybersecurity

Open ports can become entry points for attackers.

Hackers look for open ports to:

Identify running services

Find vulnerabilities

Launch attacks

For example:

An attacker finds port 22 open and tries to break into the system using **SSH (Secure Shell) brute-force login attempts.

Because of this risk, cybersecurity professionals:

Close unused ports

Monitor suspicious traffic

Configure firewalls

2. Port Scanning

Port scanning is the process of checking a computer or server to see which ports are open, closed, or filtered.

Attackers and security professionals both use this technique.

Example

A scanner sends requests to ports like:

21
22
80
443
3306

Then it checks which ones respond.

Common Port Scanning Tool

One of the most powerful tools used is:

Nmap (Network Mapper)

Security analysts use it to:

Discover open ports

Identify services

detect vulnerabilities

Example command:

nmap 192.168.1.1

This scans the target system for open ports.

Types of Port Scanning
1. TCP Connect Scan

Uses the full **TCP (Transmission Control Protocol) handshake.

Steps:

SYN

SYN-ACK

ACK

This is the most reliable scan, but it is easy to detect.

2. SYN Scan (Half-Open Scan)

Also called stealth scanning.

It sends:

SYN → waits for response

Then stops before completing the connection.

This method is commonly used in penetration testing.

3. UDP Scan

Checks ports using UDP (User Datagram Protocol).

UDP services include:

DNS (Domain Name System)

streaming

VoIP

UDP scanning is slower and harder.

3. Port Forwarding

Port forwarding is a networking technique that redirects communication requests from one port to another device or port inside a private network.

It is commonly used in:

home networks

cloud servers

gaming servers

remote access

Example of Port Forwarding

Imagine:

A router has public IP:

105.112.50.20

Inside the network:

Laptop: 192.168.1.10

If someone accesses:

105.112.50.20:3389

The router forwards the request to:

192.168.1.10:3389

This allows remote access using:

Remote Desktop Protocol

Why Port Forwarding is Risky

If configured incorrectly, attackers can exploit it.

Example risks:

exposing internal devices

unauthorized access

malware infection

Cybersecurity best practice includes:

using strong authentication

enabling VPN access

limiting exposed ports

4. Difference Between the Three Concepts
Concept	Meaning
Open Port	A port actively accepting network connections
Port Scanning	Technique used to discover open ports
Port Forwarding	Redirecting network traffic to another device
5. Real Example of a Cyber Attack

Many cyberattacks start with port scanning.

Steps:

Attacker scans network with Nmap

Finds port 21 open

Identifies service as FTP (File Transfer Protocol)

Exploits weak password or vulnerability

Gains system access

Understanding which ports attackers target first is very important in cybersecurity, ethical hacking, and SOC monitoring. Hackers usually scan common ports because they often run important services.

Top 10 Ports Hackers Target First

These ports are commonly discovered during port scanning using tools like Nmap.

1. Port 21 — FTP

Protocol: FTP (File Transfer Protocol)

Purpose: File transfer between computers.

Why attackers target it:

FTP sometimes uses weak passwords

Data is not encrypted

Attackers may:

steal files

upload malware

gain server access

2. Port 22 — SSH

Protocol: SSH (Secure Shell)

Purpose: Secure remote login to servers.

Why attackers target it:

brute-force password attacks

credential stuffing

Example attack:

Thousands of login attempts trying different passwords
3. Port 23 — Telnet

Protocol: Telnet

Purpose: Remote login.

Problem:

No encryption

Because of this, attackers can easily intercept login credentials.

Many malware families attack Telnet-enabled devices.

4. Port 25 — Email Server

Protocol: SMTP (Simple Mail Transfer Protocol)

Purpose: Sending emails.

Attackers may:

send spam

exploit mail server vulnerabilities

launch phishing campaigns

5. Port 53 — DNS

Protocol: DNS (Domain Name System)

Purpose: Converts domain names to IP addresses.

Attackers use it for:

DNS amplification attacks

data exfiltration

DNS spoofing

6. Port 80 — Web Servers

Protocol: HTTP (Hypertext Transfer Protocol)

Purpose: Websites.

Common attacks:

SQL injection

Cross-site scripting (XSS)

web server exploitation

7. Port 443 — Secure Websites

Protocol: HTTPS (Hypertext Transfer Protocol Secure)

Purpose: Encrypted web traffic.

Even though it's encrypted, attackers may still target:

vulnerable web apps

misconfigured SSL/TLS

web server exploits

8. Port 3389 — Remote Desktop

Protocol: Remote Desktop Protocol

Purpose: Remote access to Windows computers.

Very common attack target.

Attackers may:

brute-force login

deploy ransomware

control the system remotely

Many ransomware attacks start through open RDP ports.

9. Port 445 — Windows File Sharing

Protocol: Server Message Block

Purpose: File and printer sharing in Windows.

Famous cyberattack:

WannaCry ransomware exploited SMB vulnerability.

This attack spread worldwide in 2017.

10. Port 3306 — Database

Database: MySQL

Purpose: Database communication.

Attackers target it to:

steal sensitive data

access company databases

dump user information
